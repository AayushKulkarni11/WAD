Assignment4-A jQuery and jQuery Mobile Explanation
==================================================

1. Project Overview
-------------------
This project is built using:
- HTML
- jQuery
- jQuery Mobile
- small custom CSS

Pages in this project:
- `index.html`
- `departments.html`
- `login.html`
- `login_register.html`

Purpose of the project:
- create a mobile-friendly website
- use jQuery Mobile for UI layout
- use jQuery for handling form events and validation


2. Libraries Used
-----------------
`<script src="https://code.jquery.com/jquery-1.11.1.min.js"></script>`
- Loads the jQuery library
- jQuery makes JavaScript DOM selection, events, and updates easier

`<script src="https://code.jquery.com/mobile/1.4.5/jquery.mobile-1.4.5.min.js"></script>`
- Loads jQuery Mobile
- jQuery Mobile builds mobile-friendly UI components using HTML `data-role` attributes


3. What jQuery Is
-----------------
jQuery is a JavaScript library.

It is used to:
- select HTML elements easily
- handle events like click and submit
- change page content
- read and set values
- simplify JavaScript code

Without jQuery, JavaScript code is usually longer.


4. What jQuery Mobile Is
------------------------
jQuery Mobile is a UI framework built on top of jQuery.

It is used to:
- create mobile-friendly layouts
- style buttons, forms, navbars, and tables
- support touch-friendly design
- make pages responsive with less code

In this project, jQuery Mobile is used through attributes like:
- `data-role="page"`
- `data-role="header"`
- `data-role="navbar"`
- `data-role="footer"`
- `data-role="table"`
- `data-filter="true"`


5. Meaning of `$` in jQuery
---------------------------
`$` is a shortcut for `jQuery`.

Example:
`$("#email")`

This means:
- find the HTML element whose id is `email`

These two are equivalent:
`$("#email")`
`jQuery("#email")`

Why `$` is used:
- it is short
- it is easy to write
- it makes code cleaner


6. jQuery Code Used in login.html
---------------------------------
Code:

`$(document).on("pagecreate", function() {`
`    $("form").on("submit", function(e) {`
`        e.preventDefault();`
`        var email = $("#email").val();`
`        if (email) {`
`            alert("Login Successful! Welcome, " + email);`
`        }`
`    });`
`});`

Line-by-line explanation:

`$(document).on("pagecreate", function() {`
- `$(document)` selects the full document page
- `.on("pagecreate", ...)` attaches an event listener
- `pagecreate` is a jQuery Mobile event
- it runs when the jQuery Mobile page is created and ready

Why `pagecreate` is used here:
- jQuery Mobile loads and enhances page elements dynamically
- this event ensures the page UI is ready before event binding

`$("form").on("submit", function(e) {`
- selects the form element
- attaches a submit event to the form
- the function runs when the user clicks the Login button

`e.preventDefault();`
- stops the normal form submission
- prevents page refresh
- lets JavaScript control what happens

`var email = $("#email").val();`
- `$("#email")` selects the input whose id is `email`
- `.val()` reads the entered value
- the value is stored in variable `email`

`if (email) {`
- checks whether the email field is not empty

`alert("Login Successful! Welcome, " + email);`
- shows a popup message
- displays the entered email in the success message

`});`
- ends the submit event function

`});`
- ends the `pagecreate` event function

Summary of login logic:
- wait until the page is ready
- capture form submission
- stop actual submit
- read email field
- show login success message


7. jQuery Code Used in login_register.html
------------------------------------------
Code:

`$(document).on("pagecreate", function() {`
`    $("form").on("submit", function(e) {`
`        var pass = $("#password").val();`
`        var conf = $("#confirm_password").val();`
`        if (pass !== conf) {`
`            e.preventDefault();`
`            $("#error-msg").text("Passwords do not match!");`
`        } else {`
`            $("#error-msg").text("");`
`            alert("Registration Successful!");`
`        }`
`    });`
`});`

Line-by-line explanation:

`$(document).on("pagecreate", function() {`
- waits for the jQuery Mobile page creation event

`$("form").on("submit", function(e) {`
- selects the form and runs code when the form is submitted

`var pass = $("#password").val();`
- selects the password field
- gets its value
- stores it in variable `pass`

`var conf = $("#confirm_password").val();`
- selects the confirm password field
- gets its value
- stores it in variable `conf`

`if (pass !== conf) {`
- compares both passwords
- checks whether they are different

`e.preventDefault();`
- stops form submission when passwords do not match

`$("#error-msg").text("Passwords do not match!");`
- selects the element with id `error-msg`
- writes the error message into that element

`} else {`
- runs when passwords match

`$("#error-msg").text("");`
- clears any old error message

`alert("Registration Successful!");`
- shows success popup

Summary of register logic:
- wait for page creation
- listen for form submit
- compare password and confirm password
- stop submit if they are different
- show error message if invalid
- show success message if valid


8. jQuery Mobile Markup Used in Other Pages
-------------------------------------------
Even when there is no custom JavaScript, jQuery Mobile is still being used.

Examples:

`data-role="page"`
- defines one jQuery Mobile page container

`data-role="header"`
- creates a mobile-styled page header

`data-role="navbar"`
- creates a mobile navigation bar

`data-role="footer"`
- creates a page footer

`data-theme="b"`
- applies theme style `b`

`data-position="fixed"`
- keeps the footer fixed at the bottom

`class="ui-btn-active"`
- marks the current active navigation button

`class="ui-content"`
- gives jQuery Mobile content styling

In `departments.html`:

`data-role="table"`
- creates a responsive jQuery Mobile table

`data-mode="reflow"`
- makes the table adapt better on small screens

`data-filter="true"`
- enables table filtering

`data-input="#filterTable-input"`
- connects the search box to the table filter

`data-type="search"`
- styles the input as a search input

`class="ui-responsive table-stroke"`
- adds responsive and table styling

11. Definitions of Common jQuery Methods Used
---------------------------------------------
`$()`
- jQuery selector function

`.on()`
- attaches an event handler

`.val()`
- gets or sets the value of form elements

`.text()`
- gets or sets text inside an element

`alert()`
- built-in JavaScript function to show a popup

`preventDefault()`
- stops the browser's default behavior
